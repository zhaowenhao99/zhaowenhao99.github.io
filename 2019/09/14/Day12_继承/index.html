继承

继承：就是子类继承父类的属性和行为，使得子类对象具有与父类相同的属性、相同的行为。子类可以直接 访问父类中的非私有的属性和行为。如果是私有成员变量需要提供get / set 方法

extends 前面的是子类, 后面的是父类

在子类方法中访问一个变量

  • 子类局部范围找
  • 子类成员范围找
  • 父类成员范围找
  • 如果没有就报错(不考虑父亲的父亲)

好处

​ 提供代码的复用性

​ 类与类之间产生了关系,是多态的前提

坏处

​ 提高了代码的耦合性

​ 让子类独立性降低

​

super 和 this

  • 父类空间优先于子类对象产生

    在每次创建子类对象时，先初始化父类空间，再创建其子类对象本身。目的在于子类对象中包含了其对应的父类空 间，便可以包含其父类的成员，如果父类成员非private修饰，则子类可以随意使用父类成员。代码体现在子类的构造方法调用时，一定先调用父类的构造方法。

  • super 和 this 和 含义

    super ：代表父类的存储空间标识(可以理解为父亲的引用)。

    this ：代表当前对象的引用(谁调用就代表谁)。

    | 关键字 | 访问成员变量 | 访问构造方法 | 访问成员方法 |
    | :—– | ——————————- | ————————— | ———————————— |
    | this | this.成员变量 访问本类成员变量 | this(…) 访问本类构造方法 | this.成员方法(…) 访问本类成员方法 |
    | super | super.成员变量 访问父类成员变量 | super(…) 访问父类构造方法 | super.成员方法(…) 访问父类成员方法 |

构造方法

构造方法的名字是与类名一致的。所以子类是无法继承父类构造方法的。 

构造方法的作用是初始化成员变量的。所以子类的初始化过程中，必须先执行父类的初始化动作。子类的构 造方法中默认有一个 super() ，表示调用父类的构造方法，父类成员变量初始化后，才可以给子类使用

方法重写

方法重写：
子类中出现与父类一模一样的方法时（返回值类型，方法名和参数列表都相同），会出现覆盖效果，也称为重写或者复写。声明不变，重新实现。 

注意事项

1. 私有方法不能被重写
2. 子类方法访问权限不能更低(public > 默认 > 私有)
3. 开发的时候要保持一致

方法重写和方法重载的区别
